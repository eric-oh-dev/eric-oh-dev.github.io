JS Fundamentals

JavaScript and it's relationship to HTML and CSS

11 April 2025

Learning to code is like trying to speak a new language with three different dialects - HTML, CSS, and JavaScript. I had no idea how they all worked together or what things like "control flow" or "the DOM" meant.
Here's how I've come to understand them - and how I'd explain them to a friend who's curious about what I've been learningl

HTML, CSS, and JavaScript: The Holy Trinity of the Web

A website is like a house
- HTML is the structure - the walls, doors, and layout of your house.
- CSS is the paint and furniture - it makes everything look nice.
- JavaScript is the electricity and plumbing - it makes things do stuff, like turning on lights when you press a switch.
Without JavaScript, your website might look great, but it won't be interactive. With JavaScript, buttons can respond when clicked, forms can be validated, and pages can update without refreshing.

Control Flow & Loops - Like Making a Cup of Tea

Control flow is the order in which your code runs - kind of like a recipe. Sometimes you do things stop by stop, sometimes you make decisions (if statements), and sometimes you repeat actions (loops).
1. Boil water
2. If you have a teabag, put it in the cup -> else, go to the shop and buy teabags
3. pour hot water into the cup
4. Wait 3 minutes
5. Add milk if you want it
6. Stir and enjoy!
Now imagine doing this for 5 people - you'd probably use a loop!

The DOM: Your Website's Blueprint That JavaScript Can Change

The DOM (Document Object Model) is like a live map of your website's HTML. JavaScript can use this map to read, change, or delete things on the page.

Arrays vs Objects: Boxes and Labelled Drawers

An array is like a row of boxes, where each box has a number. You use the number (index) to get the item.

An object is more like a drawer with labels - you use a name (key) to access each item.

Use arrays when order matters, and objects when you need key-value pairs.

What Are Functions and Why Do We Use Them?

Functions are like little machines or routines. You give them instructions, and then you can reuse those instructions anytime without rewriting them.

Instead of writing "Hello, ___!" over and over again, you create one function, and then just call it when you need it. Functions help make your code reusable, clean, and easier to read.
